# Transcript Tool

A lightweight tool combining transcript viewing, freeform annotation, and test case seeding. Generated by the skill as part of each run — no separate install required.

## What It Does

Renders each trial's **full message stream** from `result.json` in whichever standard format the runner stored it (OpenAI Chat Completions, OpenAI Responses items, or Anthropic Messages): user input, every assistant turn, tool calls with arguments, tool outputs, and the final reply. Works for any agent; no agent-specific logic.

The user writes a freeform comment on any trial:
- "This is wrong — it processed the refund without checking identity first"
- "Looks good but too verbose"
- "This is exactly right"

An LLM parses the comment into a pass/fail judgment, the specific failure mode or success pattern, and any implicit criteria that can become a case assertion or rubric refinement.

## Three Uses in One

**Reviewing** — read what the agent actually did. Essential after any 0% or unexpected result.

**Annotating** — annotations accumulate as human ground truth. Used to calibrate rubrics: compare annotations against LLM rubric scores and adjust where they diverge.

**Seeding** — after annotating, the tool asks: _"Should I add this as a test case?"_ If yes, the input + annotations become an EvalCase with assertions derived from the comment.

## Persistence

Annotations are saved automatically on every submit to `runs/<YYYY-MM-DD>/annotations.jsonl` — one record per annotation:

```
{
  run_file:    string,   # path to the result file this annotation belongs to
  case_id:     string,
  trial:       int,
  comment:     string,   # raw freeform text
  judgment:    "pass" | "fail" | "unclear",   # LLM-extracted
  failure_mode: string | null,                # LLM-extracted
  implicit_criteria: List[string],            # LLM-extracted; candidates for assertions or rubric refinement
  seed_case:   bool,     # whether user confirmed this should become a test case
  annotated_at: string   # ISO timestamp
}
```

The tool must confirm to the user after each save: _"Saved."_ Annotations persist across sessions and are reused for rubric calibration and test case seeding.

## Design Principles

- **Freeform over forms** — one text field, not a rubric to click through. Structure is extracted after, not imposed before.
- **Skip everything** — annotations are optional. Viewing alone is valuable.
- **Lightweight** — generated as a local HTML file or started with one command as a local server.
